## 📌 模块概述
**目标**: 基于《[一个海量在线用户即时通讯系统（IM）的完整设计Plus](https://mp.weixin.qq.com/s/TYUNPgf_3rkBr38rNlEZ2g)》实现 **S2C消息推送机制 (Server-to-Client)**，对应文章第1.2.2.5节的 **推送流程**。

> 📖 **架构参考**: 文章第1.2.2.5节 - "1.业务线调用push数据接口 → 2.Logic向redis检索目标用户状态 → ... → 6.Gate向用户推送数据"。

## ✨ 实现效果
- [ ] Logic层可以主动推送消息给在线用户
- [ ] 支持单推（P2P）和群播（Group Cast）
- [ ] 推送失败时自动降级为离线消息（写入Kafka）
- [ ] 推送状态可追踪（已发送/已送达/已读）
- [ ] 支持推送重试机制（最多3次）
